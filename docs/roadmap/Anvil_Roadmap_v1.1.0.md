## 📌 Phase 5: 전략적 협업 (Dex & Sophia) - "지능형 데이터 서빙" [NEW]

Vortex(소피아 누님)가 학습에만 집중할 수 있도록, Anvil은 **'수익 확률이 높은 종목'**만 골라서 떠먹여 주는 스마트한 서빙 기능을 탑재한다.

### 5.1 주간 Value 후보 발굴 파이프라인 (Sunday Pipeline)
- **일정**: 매주 일요일(장 개장 전) 가동.
- **로직**: 일봉 데이터를 풀스캔하여 저평가 구간 또는 특정 패턴(예: 역배열 돌파, 바닥권 매집)을 보이는 '이번 주의 Value 후보 Top 10' 선정.
- **연동**: 선정된 심볼 리스트를 Postgres `value_tier_tickers` 테이블에 자동 업데이트하여 소피아 누님이 우선순위를 두게 함.

### 5.2 종목 성격별 피처 차별화 (Dynamic Features)
- **전략**: 우량주(AAPL, MSFT)와 급등주(NVDA, TSLA)의 지표 반응 속도 차이를 데이터로 증명.
- **협업**: 앤빌(Anvil)이 종목별 변동성 등급을 Vortex(ML)에 넘겨주면, 소피아(Sophia) 누님이 모델 추론 시 가중치를 조절하도록 설계.

## 📌 Phase 6: 다이내믹 유니버스 (Screener) - "종목은 걸러지는 것이다" [NEW]

하드코딩된 리스트를 넘어, 시장의 에너지가 쏠리는 곳을 실시간으로 추적하는 **'이오츠의 레이더망'**을 구축한다.

### 6.1 이오츠/카포의 레이더망 (screener.py)
- **핵심 필터링 조건**:
    - **거래대금(Capo's Rule)**: 최근 20일 평균 거래대금 $1,000만 이상 (유동성 확보).
    - **변동성(Capo's Rule)**: 최근 14일 ATR 비율 3% 이상 (단타 수익 구간 확보).
- **로직**: S&P 500 및 나스닥 전체 종목 스캔 후 상위 100개 종목 추출.

### 6.2 동적 유니버스 관리 (dynamic_universe)
- **적재**: Postgres `dynamic_universe` 테이블에 날짜별 타겟 티커 저장.
- **자동화**: ETL 추출기 가동 시 하드코딩된 리스트 대신 `dynamic_universe`를 참조하여 데이터 수집의 효율성과 수익성을 극대화.

## 📌 Phase 7: 인프라 고도화 및 데이터 품질 감시 [NEW]

### 7.1 데이터 무결성 검증 (Validation) 강화
- **자동 탐지**: 결측치(Null), 이상치(Outlier), 타임스탬프 역전 현상을 감지하여 리포트에 기록.
- **자가 치유**: 데이터 누락 발견 시 해당 구간만 자동으로 Alpaca 재요청 및 패치(Patch) 로직 구현.

### 7.2 DuckDB 연동 및 로컬 분석 최적화
- **하이브리드 스토리지**: 영구 저장은 Postgres, 고속 분석 및 임시 처리는 DuckDB를 활용하여 분석 속도를 10배 이상 향상.
